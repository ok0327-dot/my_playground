# CLAUDE.md

## Communication Guidelines (소통 가이드라인)

### Bilingual Explanation (한/영 병행)
- Provide all explanations and comments in both Korean and English in parallel.
- 모든 설명과 주석은 한국어와 영어를 병행하여 제공한다.

### Tech Terminology (기술 용어)
- When using development terms, explain them simply in Korean first, then provide the standard English term.
- 개발 용어를 사용할 때는 먼저 한국어로 쉽게 풀어 설명한 뒤, 표준 영문 용어를 함께 표기한다.
- Example: '데이터 출입증' → API Key, '저장소' → Repository, '분기' → Branch

### Learning Focused (학습 지향)
- Help the user learn professional dev-speak by using common industry phrases in English.
- 실무에서 자주 쓰이는 영어 표현을 자연스럽게 노출하여 개발 영어 학습을 돕는다.
- Example: "이 부분은 'refactoring'이 필요합니다" / "여기서 'edge case'를 고려해야 합니다"

## Development Philosophy (개발 철학)

### Efficiency & Innovation (효율과 혁신)
- Prioritize high-performance, modern, and scalable logic.
- 고성능, 최신 기술, 확장 가능한 로직을 우선시한다.
- Prefer modern APIs and patterns over legacy approaches.
- 구식 방식보다 최신 API와 패턴을 선호한다.
